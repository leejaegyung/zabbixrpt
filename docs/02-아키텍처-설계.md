# 02. 아키텍처 & 기술 설계 (개발 단계)

## 1. 기술 스택

| 레이어 | 기술 | 비고 |
|--------|------|------|
| 프런트엔드 | Vue 3 (Composition API) + Vite + Tailwind CSS | React → Vue 1:1 포팅 |

| 웹서버 | nginx + php-fpm | |
| 컨테이너 | Docker Compose | 멀티스테이지 빌드, 폐쇄망 동봉 |
| 테스트 | Vitest (프런트), Pest/PHPUnit (백엔드) | 핵심 로직 ≥70% |

## 3. Laravel 프록시 설계

### 엔드포인트
`POST /api/zabbix` — 본문: `{ method, params }` (클라이언트는 토큰/인증방식 모름)

### `ZabbixClient::call($method, $params)` — 버전 자동감지 인증
1. **1차**: `Authorization: Bearer {token}` 헤더 + body에 `auth` 없음 (Zabbix 7.0+/8.0).
2. 응답이 `error.code === -32602` 이고 메시지에 auth/session/token 포함 → **2차 폴백**.
3. **2차**: body에 `auth: {token}` 포함, Authorization 헤더 제거 (Zabbix ≤6.0 legacy).
4. 결과를 정규화: 성공 `{ result }`, 실패 `{ error: { code, message, normalized } }`.
5. 에러 정규화 매핑(기존 클라이언트 문구 유지): NETWORK/CORS, HTTP_404, HTTP_401, NOT_JSON, AUTH_EXPIRED.

### 보안
- 토큰은 `.env`(`ZABBIX_API_TOKEN`) 또는 서버 세션 보관. 클라이언트 응답/`localStorage`에 평문 노출 금지.
- 허용 메서드 화이트리스트: `host.get`, `problem.get`, `trigger.get`, `item.get`, `history.get`.
- 멀티유저 대비: 요청별 토큰 전달 옵션(헤더)도 설계상 허용하되 기본은 서버 보관.

## 4. React → Vue 포팅 매핑

| 기존 (React, zabbix8.0.html) | 신규 (Vue) |
|------|------|
| `useState` 다수 | Pinia stores (connection / data / viewSettings / advanced) |
| `callZabbixApi` (Bearer→legacy 폴백) | `api/zabbix.js` → Laravel 프록시 (폴백은 서버로 이동) |
| `getAnalysisData`, `calculateStoragePrediction`, `getAverageValue` | `composables/useAnalysis.js` (순수 함수, 단위 테스트 대상) |
| `exportPDF` / `executeExport` | `composables/usePdfExport.js` |
| `formatItemValue`, severity, 시간 헬퍼 | `composables/useFormat.js` |
| `renderAnalysisSection` (7 카드) | `AnalysisSection.vue` + 카드 컴포넌트 |
| `renderHostsWeb/ProblemsWeb/ItemsWeb` | `HostsTable/ProblemsTable/ItemsGrid.vue` |
| `ErrorBoundary` (8.0) | `onErrorCaptured` 기반 ErrorBoundary 컴포넌트 |
| 커스텀 alert/confirm 모달 | `common/Modal.vue` + 모달 store |
| `localStorage` 영속(설정) | 동일 (단 토큰은 제외) |

## 5. Docker 구성 (폐쇄망)

- **멀티스테이지**: `node` 스테이지에서 `npm ci && vite build` (의존성 lock 동봉) → 정적 산출물을 php/nginx 스테이지로 복사.
- **composer**: `--no-dev --optimize-autoloader`, vendor 동봉.
- 런타임 외부 네트워크 0 — 빌드 시점에 모든 패키지 확보(오프라인 빌드 검증 포함).
- 서비스: `nginx`(80) → `php-fpm`(Laravel) → 외부 Zabbix(`ZABBIX_API_URL`).
- 환경변수: `ZABBIX_API_URL`, `ZABBIX_API_TOKEN`(선택), `APP_KEY`.

## 6. 미해결/착수 전 확인

- Zabbix 테스트 서버(6.0/7.0/8.0) 접근 — 인증 폴백 통합 테스트에 필요.
- 토큰 보관 정책: 단일 고정 토큰(.env) vs 요청별 토큰 — 운영 시나리오 확인.
- 기존 HTML 회귀 기준 스크린샷 확보(검증 단계 입력).
