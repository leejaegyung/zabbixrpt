# 05. 설정 정보구조(IA) — 조회 콘솔 / 설정 드로어 분리

## 배경
좌측 사이드바에 운영용(데이터 조회)과 설정용(환경설정)이 섞이고, 설정이 사이드바 패널과 헤더 모달로
쪼개져 두서가 없었다. 운영 흐름과 환경설정을 분리해 체계화한다.

## 확정 구조

### 사이드바 = "조회 콘솔" (운영 흐름)
- 연결: Zabbix API URL, API Token
- 기간: From / To (날짜+시간)
- 조회: 전체 갱신 + 호스트/문제/아이템 개별 조회
- 컴포넌트: `ApiSettingsPanel.vue` (유지)

### 헤더 톱니 → 설정 드로어 (우측 슬라이드) = 환경설정 통합
`SettingsDrawer.vue` — 4개 섹션 스택(전부 표시, 스크롤):
1. **Display** — 섹션 토글(분석/호스트/문제/아이템) + 7대 분석 개별 토글
2. **Report** — 커스텀 로고 / 워터마크 / 그래프 크기 / 저장·적용
3. **Profile** — Export / Import (토큰 제외)
4. **Sandbox** — 가상 대시보드 로드 (개발·테스트)

## 변경 요약
| 항목 | 변경 |
|------|------|
| 신규 | `components/SettingsDrawer.vue` (우측 드로어, Esc/백드롭 닫기, 슬라이드 트랜지션) |
| 제거(흡수) | `ViewSettingsModal.vue`, `panels/AdvancedSettingsPanel.vue` |
| 스토어 | `ui.isViewSettingsOpen` → `ui.isSettingsOpen` |
| App | 사이드바=ApiSettingsPanel만; 헤더 톱니→드로어; SANDBOX 인라인 카드 제거 |
| 영향 없음 | PDF 보고서(설정은 웹 전용, 스토어 값만 읽음), 핵심 로직/컴포저블 |

## 검증
- Vitest 30/30 통과.
- 사이드바: Connection만(Report/Sandbox 제거) 확인.
- 드로어: Display/Report/Profile/Sandbox 4섹션 렌더, 드로어 내 "가상 대시보드 로드" → 데이터·분석 렌더 확인. 에러 0.
- PDF 6페이지 흰 종이 유지(회귀 없음).
